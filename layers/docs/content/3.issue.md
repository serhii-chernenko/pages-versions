# Issue explanation

1. The `findMany` method is used to fetch the pages with relations to versions and content.
2. But I'm expection possible sorting by `createdAt` for `pages` but the field is actually existing only in the `page_versions` table.
3. Also I want to have a possibility to search by the `title` and `slug` fields. But they are existing only in the `page_version_contents` table.
4. Also, I want to have a possibility to sort by different fields. E.g., `createdAt`, `title`, `slug`, etc. But the `createdAt` field is existing only in the `page_versions` table, and the `title` and `slug` fields are existing only in the `page_version_contents` table.

I see two possible solutions:

1. Duplicate the `createdAt` field in the `pages` table. But it doesn't solve the problem with searching by `title` and `slug`. And it will be hard to keep the data in sync.
2. Make a few requests to get the data. But it affects the performance and increases usage costs of the database. Honestly, I don't even understand how to correctly combine the data from different tables in case of `limit` and `offset` usage for the pagination.

I kept `TODO` comments in the code to show where I need to implement the solution.
